Later.js

The simplest way to work with recurring schedules.

Easily define complex schedules and then quickly calculate future or previous schedule occurrences. Works in Node and in the browser.

View on GitHub Download

Large library of time periods

Schedules can be produced with all common time periods (seconds, minutes, etc) as well as ISO weeks, day of year, nth week of month, and others.

Zero dependencies

Later has exactly zero dependencies which, along with its compact size (about 8kb compressed), makes incorporating it into your application a piece of cake.

Full Cron support

Create a Later schedule from any valid Cron expression and easily calculate future occurrences.

Fully extensible

Create your own time periods that can then be used in any schedule. Write modifiers to add additional functionality to existing time periods.

Well tested

Supported by a test suite of over 40,000 unit and scenario tests to help ensure it produces correct results.

Support for English expressions

All schedules can be defined using intuitive English phrases such as 'every 5 mins' making it easy for non-technical people to define schedules.

Quick start

Step 1: Define a schedule

The first step is to create a new schedule definition. There are a bunch of different ways to define a schedule, but at the end of the day they are just simple json objects that can easily be serialized and stored.

  // define a new schedule
  var text = later.parse.text('at 10:15am every weekday');
  var cron = later.parse.cron('0 0/5 14,18 * * ?');
  var recur = later.parse.recur().last().dayOfMonth();
  var manual = {schedules: [{M: 3, D: 21}]};

Schedules can also get as complicated as you need:

  // this schedule will fire on the closest weekday to the 15th
  // every month at 2:00 am except in March
  var complex = later.parse.recur()
                  .on(15).dayOfMonth().onWeekday().on(2).hour()
                .and()
                  .on(14).dayOfMonth().on(6).dayOfWeek().on(2).hour()
                .and()
                  .on(16).dayOfMonth().on(2).dayOfWeek().on(2).hour()
                .except()
                  .on(3).month();

Step 2: Configure timezone

All schedule definitions are completely timezone agnostic. When you need to calculate occurrences, you can decide to perform the calculation using local time or UTC.

  // set later to use UTC (the default)
  later.date.UTC();

  // set later to use local time
  later.date.localTime();

Step 3: Calculate occurrences

Once a schedule has been defined, calculating future (or past) occurrences is very straightforward. You can calculate as many occurrences as you need in either a forward or backward direction.

  // calculate the next 10 occurrences of the text schedule
  var next = later.schedule(text).next(10);

  // calculate the previous occurrence starting from March 21, 2013
  var next = later.schedule(text).prev(1, new Date(2013, 2, 21));

Step 4: Execute code using a Later schedule

Later includes implementations of setTimeout() and setInterval() that work with Later schedules instead of milliseconds. This makes it extremely easy to execute code based on a schedule definition.

  // execute fn one time on the next occurrence of the text schedule
  var timer = later.setTimeout(fn, text);

  // execute fn for each successive occurrence of the text schedule
  var timer2 = later.setInterval(fn, text);

  // clear the schedule when you are done
  timer2.clear();
